## Non-negotiable guardrails

- No revenue guarantee: every paid motion must be validated with real leads, contracts, or checkout tests.
- No secrets in git: API keys, webhook secrets, payout IDs, bank data, and live payment keys belong only in provider secret stores.
- Real payout/revenue-account connection cannot be completed by code alone; it requires authenticated provider dashboards, business verification, and sensitive bank information.
- Archived repositories stay read-only unless explicit action-time confirmation is given to unarchive or deploy.
- Regulated domains keep strict no-medical/no-financial-advice/no-safety-guarantee boundaries.
